1 클론 & 환경 준비

GoClaw 레포를 클론하고 시크릿을 자동 생성합니다.

git clone -b main https://github.com/nextlevelbuilder/goclaw.git
cd goclaw

# 시크릿 자동 생성 (게이트웨이 토큰 + 암호화 키)
chmod +x prepare-env.sh && ./prepare-env.sh

.env 편집

# PostgreSQL 비밀번호
POSTGRES_PASSWORD=your_secure_password

# OpenClaw(18789/18790)과 포트 충돌 방지
GOCLAW_PORT=18800
POSTGRES_PORT=5433

# 대시보드 전체 접근 권한을 위해 owner 설정
GOCLAW_OWNER_IDS=admin,system
⚠ GOCLAW_OWNER_IDS를 빠뜨리면 대시보드에서 에이전트 수정 시 퍼미션 에러가 발생합니다.

2 GoClaw 기동

두 가지 방법 중 선택합니다. Max 구독으로 Claude CLI를 쓰려면 소스 빌드가 필요합니다.

방법 A — 사전 빌드 이미지 (가장 빠름)
docker compose -f docker-compose.yml -f docker-compose.postgres.yml up -d
방법 B — 소스 빌드 (Claude CLI 포함)
# Claude CLI 포함 (Max 구독 사용자용)
make up WITH_CLAUDE_CLI=1

정상 동작 확인

curl http://localhost:18800/health
# {"status":"ok","protocol":3}

DB 마이그레이션

docker compose -f docker-compose.yml -f docker-compose.postgres.yml \
  -f docker-compose.upgrade.yml run --rm upgrade

3 LLM 프로바이더 설정

GoClaw은 6종의 프로바이더를 지원합니다. Claude Max 구독 사용자는 A 또는 B를 선택하세요.

Claude CLI
claude_cli
소스 빌드 필요. CLI를 subprocess로 실행하여 Max 구독 OAuth 토큰을 사용합니다.
구독만
claude-max-api-proxy
openai_compat
별도 Node.js 컨테이너로 프록시 운영. 빌드 불필요. sed로 0.0.0.0 바인딩 패치 필요.
구독만
Anthropic Native
anthropic_native
직접 HTTP+SSE 호출. 프롬프트 캐싱, Extended Thinking 지원.
종량제
OpenAI-Compatible
openai_compat
OpenRouter, Groq, DeepSeek, Mistral 등 호환 엔드포인트.
종량제
ACP
acp
Agent Client Protocol. subprocess + MCP 브릿지 방식.
구독만
DashScope
dashscope
Alibaba Qwen. Extended thinking budget 지원.
종량제
ℹ Claude CLI 모델명은 별칭을 사용합니다: opus, sonnet, haiku (claude-opus-4가 아님)

Claude CLI 프로바이더 등록 예시

curl -s -X POST \
  -H "Authorization: Bearer $GATEWAY_TOKEN" \
  -H "X-GoClaw-User-Id: admin" \
  -H "Content-Type: application/json" \
  http://localhost:18800/v1/providers \
  -d '{
    "name": "claude-cli",
    "provider_type": "claude_cli",
    "api_key": "",
    "enabled": true,
    "settings": { "default_model": "opus" }
  }'

4 에이전트 생성

GoClaw 채팅 API가 동작하려면 agent_key: "default" 에이전트가 필수입니다.

curl -s -X POST \
  -H "Authorization: Bearer $GATEWAY_TOKEN" \
  -H "X-GoClaw-User-Id: admin" \
  -H "Content-Type: application/json" \
  http://localhost:18800/v1/agents \
  -d '{
    "agent_key": "default",
    "display_name": "Default Agent",
    "agent_type": "open",
    "provider": "<PROVIDER_ID>",
    "model": "opus",
    "is_default": true,
    "enabled": true
  }'

5 워크스페이스 파일 이전

GoClaw open 에이전트는 부트스트랩 파일을 DB에 저장합니다. 디스크가 아닙니다.

⚠ OpenClaw 워크스페이스 파일을 디스크에 복사해도 에이전트가 읽지 않습니다. 반드시 WebSocket RPC 또는 PostgreSQL을 통해 DB에 넣어야 합니다.

아키텍처 비교

OpenClaw (이전): Node.js → 파일 기반 상태 → ACP/subagents → Claude CLI ├→ 멀티 에이전트 (agents.list) ├→ 멀티유저 (session.dmScope) └→ 봇 간 위임 (subagents, sessions_spawn) GoClaw (이후): Go 바이너리 → PostgreSQL → 멀티테넌트 → Claude CLI 프로바이더 ├→ Web Dashboard (React 19) ├→ 에이전트 팀 & 위임 (칸반 보드 + 메일박스) ├→ per-user 세션 격리 (PostgreSQL) ├→ Model Steering (Track/Guard/Hint) ├→ Knowledge Graph + Extended Thinking └→ Desktop Edition (GoClaw Lite)

트러블슈팅

agent not found: default

agent_key: "default" 에이전트를 생성하세요. GoClaw HTTP 채팅 API에 필수입니다.

대시보드 "Failed to update agent"

.env에 GOCLAW_OWNER_IDS=admin,system을 설정하고 재시작. owner 역할이 필요합니다.

Claude CLI "unsupported model"

모델 별칭 사용: opus, sonnet, haiku (claude-opus-4가 아님).

Claude CLI 401 인증 에러

OAuth 토큰(sk-ant-oat01-...)은 CLI를 통해서만 작동. API 키로 직접 사용 불가.

Slack 봇이 연결 안 됨

OpenClaw 중지 확인. Bot 토큰 xoxb-, App 토큰 xapp- 확인.

부트스트랩 파일이 로드 안 됨

GoClaw open 에이전트는 DB에서 읽음. agents.files.set RPC 또는 agent_context_files 테이블 직접 삽입.

macOS tar "Permission denied"

COPYFILE_DISABLE=1로 확장 속성 건너뛰기. --privileged Alpine 컨테이너 사용.
